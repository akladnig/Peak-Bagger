Primary flow:
1. User launches app for first time
2. App queries Overpass API for Tasmanian peaks
3. Peaks stored in ObjectBox database
4. Map displays firebrick-colored triangles for each peak (W3C #B22222)
5. User hovers over peak → tooltip shows name + elevation
6. User types in search box → dropdown shows matching peaks
7. User selects peak → map centers on peak

Alternative flows:
- First launch: No internet → show error, allow retry
- Subsequent launches: Load peaks from ObjectBox (skip API)
- Refresh button: Re-query Overpass API, update database

- API failure: Show error message, allow retry

**Data Schema:**
```
Peak (Entity):
  - id: int (auto-generated by ObjectBox)
  - name: String (from OSM "name" tag, default to "Unknown" if missing)
  - elevation: double? (meters, from OSM "ele" tag - may be string like "1234m")
  - latitude: double (from OSM "lat" attribute or center lat)
  - longitude: double (from OSM "lon" attribute or center lon)
  - area: String? (populated in future feature)
```

**Overpass API Response Format:**
- Nodes returned as JSON with `elements` array
- Each element has: `type`, `id`, `lat`, `lon`, `tags` (contains name, ele)
- Tags are stored in `tags` object: `{ "name": "Mount Wellington", "ele": "1271" }`
- Use `out center;` to get center coordinates (includes `center` object with lat/lon)
- Elevation "ele" tag may be in meters (string) - parse to double
- Example: `{"type":"node","id":123456,"lat":-42.8828,"lon":147.3318,"tags":{"name":"Mount Wellington","ele":"1271"}}`

**Overpass Query:**
```
{{geocodeArea:Tasmania}}->.searchArea;
node["natural"="peak"]["name"]["ele"](area.searchArea);
out center;
```

**Error Handling:**
- Network failure on first launch: Show error, offer retry button
- Invalid peak data: Skip and continue, log warning
- Empty database on load: Trigger initial fetch

**Edge Cases:**
- Peak without elevation: Store with null elevation, display "Unknown" in tooltip
- Peak without name: Store with default name "Unknown"
- Very large dataset: Paginate or limit to reasonable number
- Search with no results: Show "No peaks found" message
</requirements>

<boundaries>
Edge cases:
- No internet on first launch: Show error dialog with retry option
- API returns malformed data: Validate and skip invalid entries
- Duplicate peak names: Allow duplicates, use unique IDs
- Search by elevation: Support partial match (e.g., "1000" matches "1100m")

Error scenarios:
- Overpass API timeout: Show "Request timed out" after 30 seconds
- Empty response: Show "No peaks found in Tasmania"
- Database write failure: Show error, don't crash

Limits:
- Peaks displayed at zoom level 12+, do not display peaks at zoom < 12
- Initial load: Maximum 1500 peaks to prevent memory issues
- Search results: Maximum 20 results in dropdown
</boundaries>
